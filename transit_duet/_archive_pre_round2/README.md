# Archive: pre-round-2 results

This directory holds artefacts from the round-1 submission that are no longer
referenced by the round-2 paper (`paper/sections/5_experiments.tex` Tables I/II
and the figures in `paper/figures/`). Kept for provenance / audit only — do
**not** read these for any current claim.

## Why archived

* **`logs_remote/A_full_seed{42,123,456}/`** — 300-episode runs from the v2
  channels-mode runner (`runner_v2.py`). These were the basis of the round-1
  Table II `A_full` row. The round-2 paper was rewritten around HIRO
  goal-conditioned coupling (`runner_v3.py` with `coupling_mode: hiro`),
  trained as `H_hiro_seed*`. A_full numbers do **not** correspond to any row
  in the round-2 paper, and the upstream `make_result_figures.py` no longer
  reads from them.

* **`eval_per_ckpt/H_tpc_alpha/`** — per-checkpoint eval for an alpha-cap
  tuning sweep (`H_tpc_alpha`) that was tried but dropped during round-2
  finalisation. The retained ablation rows are the seven entries in Table II
  (`H_hiro`, `-TPC`, `-MORL`, `-CSBAPR`, `-DemNoise`, `-Hindsight`, `-HoldFB`,
  `-Elastic`); `H_tpc_alpha` is not one of them.

* **`results_remote/`** — round-1 aggregate CSVs (`ablation.csv`,
  `baselines.csv`, `composite.csv`, `pareto.csv`, `summary.txt`). All values
  are stale: they were generated by `composite_score.py` before the
  per-episode-aggregation fix (round-2 P0-9), and they reference the legacy
  ablation names (`A_full`, `B_no_holding_feedback`, ...) rather than the
  current `H_hiro_*` set.

## What replaces these (round-2 sources of truth)

| Old (here, archived)                  | New (live, paper)                                 |
|---------------------------------------|---------------------------------------------------|
| `logs_remote/A_full_seed*`            | `logs/H_hiro_seed*` (HIRO main result, runner_v3) |
| `eval_per_ckpt/H_tpc_alpha`           | `eval_per_ckpt/H_hiro` + 7 ablation dirs          |
| `results_remote/composite.csv`        | `results_remote/round2_composite.csv` (regen by `scripts/run_paper_round2.sh`) |

## How to reproduce round-2 numbers

From the project root (`transit_duet/`):

```bash
bash scripts/run_paper_round2.sh           # full pipeline (~14h on 1 GPU)
bash scripts/run_paper_round2.sh --quick   # 50-episode sanity sweep
```

Outputs land in `logs/<exp>_seed<N>/`, `logs/eval_per_ckpt/<exp>/`, and
`paper/figures/`.
